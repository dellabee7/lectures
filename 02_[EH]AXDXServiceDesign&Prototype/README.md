## 3. [부록 1] 2주차 실습: 프로젝트 캔버스 서식

창업 아이디어의 핵심 가치를 한 페이지로 정의하기 위한 도구입니다.

> ### **AX/DX Startup Project Canvas**
> 
> - **Project Vision**: 서비스가 지향하는 궁극적인 목표와 가치
>     
> - **Target Customer**: 페인 포인트를 가진 구체적인 고객 세그먼트
>     
> - **Core Problems**: 해결하고자 하는 시장의 3가지 핵심 문제
>     
> - **AX Solution**: AI/DX 기술이 문제를 해결하는 구체적인 방식 (예: AI 에이전트, RAG 등)
>     
> - **Unique Value**: 경쟁사 대비 우리만의 압도적 차별점
>     
> - **Key Metrics**: 성공을 측정할 수 있는 데이터 지표 (KPI)
>     
> - **Resource & Tech Stack**: 필요한 인적 자원 및 기술 스택 (Claude, Figma 등)
>

## 4. [부록 2] 창업팀 구성 역할별 체크리스트 (RACI 연계)

창업 초기 팀 빌딩 시 각 역할자가 반드시 체크해야 할 항목입니다.

- **PM (Project Manager)**:
    
    - [ ] 전체 일정(WBS)과 마일스톤이 공유되었는가?
        
    - [ ] 팀 내 커뮤니케이션 채널(Slack/Notion)이 활성화되었는가?
        
- **PO (Product Owner)**:
    
    - [ ] 비즈니스 모델(BMC)의 수익성이 검증되었는가?
        
    - [ ] 백로그 우선순위가 시장 가치 중심으로 정렬되었는가?
        
- **UX/UI Designer**:
    
    - [ ] 사용자 여정 지도(Journey Map)를 통해 페인 포인트를 도출했는가?
        
    - [ ] Figma 프로토타입이 실제 구현 가능하도록 설계되었는가?
        
- **Developer (FE/BE/AI)**:
    
    - [ ] 제안된 AX 솔루션의 기술적 실현 가능성을 검토했는가?
        
    - [ ] MVP 구현을 위한 API 및 데이터베이스 구조를 수립했는가?
        
- **QA / Analyst**:
    
    - [ ] 시장 예측 데이터(TAM/SAM/SOM)의 근거가 타당한가?
        
    - [ ] 최종 결과물의 안정성을 테스트할 시나리오를 갖추었는가?
