# [강의 요약] 경영정보시스템(MIS) 10주차: EA 이행 및 거버넌스

## 0. 리딩 메시지 (Core Insight)
> [cite_start]"EA의 실질적인 성공은 화려한 설계도가 아닌 '지속적인 이행(Implementation)'과 '엄격한 거버넌스(Governance)'에 달려 있다. 거버넌스가 없는 아키텍처는 박제된 문서일 뿐이며, 비즈니스 변화에 따라 실시간으로 업데이트되는 동적 체계가 필수적이다." [cite: 5210]

---

## 1. EA 이행 전략 (Implementation Strategy)
목표 아키텍처(To-Be)를 실현하기 위한 단계별 전환 로드맵을 수립하는 과정입니다.

* [cite_start]**점진적 전환 전략 (Phased Migration)**[cite: 5210]: 
    * [cite_start]리스크 분산을 위해 업무 중요도와 기술적 시급성에 따라 단계별로 시스템을 이행합니다. [cite: 5210]
* [cite_start]**Gap 분석 및 로드맵 수립**[cite: 5210]: 
    * [cite_start]현행(As-Is)과 목표(To-Be) 간의 기술적/비즈니스적 격차를 식별합니다. [cite: 5210]
    * [cite_start]데이터 이관(Migration), 신규 구축, 기존 시스템 폐기 일정을 통합한 마스터 플랜을 도출합니다. [cite: 5210]

---

## 2. EA 거버넌스 체계 (Governance Framework)
[cite_start]EA의 표준을 유지하고 프로젝트 준수 여부를 통제하는 의사결정 구조입니다. [cite: 5210]

* [cite_start]**아키텍처 위원회 (Architecture Board)** [cite: 5210][cite_start]: 전사 기술 표준 승인 및 주요 아키텍처 의사결정을 담당하는 최상위 기구입니다. [cite: 5210]
* [cite_start]**준수성 검토 (Compliance Review)** [cite: 5210][cite_start]: 모든 IT 프로젝트가 전사 표준을 준수하는지 단계별(착수/종료)로 심의합니다. [cite: 5210]
* [cite_start]**예외 관리 (Waiver Process)** [cite: 5210][cite_start]: 비즈니스 특수성으로 인한 표준 미준수 시, 공식적인 승인 절차를 통해 예외 사항을 관리합니다. [cite: 5210]

## 3. [실습 도구] EA 성숙도 및 준수성 진단표
[cite_start]팀 프로젝트 시 대상 기업의 EA 수준을 평가하기 위한 체크리스트입니다. [cite: 5210]

### 가. EA 거버넌스 성숙도 진단 (Maturity Focus)
| 진단 항목 | 점검 내용 | 자가 점검 (1~5) |
| :--- | :--- | :---: |
| **의사결정 체계** | EA 관련 최상위 의사결정 기구(위원회)가 실질적으로 작동하는가? | |
| **표준 준수성** | 모든 IT 프로젝트 추진 시 EA 표준 준수 여부를 공식적으로 검토하는가? | |
| **인적 역량** | 각 도메인(BA, DA, AA, TA)별 전문 아키텍트가 확보되어 있는가? | |
| **관리 도구** | 아키텍처 리포지토리(EAM 도구)를 통해 산출물이 최신화되고 있는가? | |

### 나. 프로젝트 EA 준수성 체크리스트 (Compliance Focus)
* [cite_start][ ] **비즈니스 정렬**: 해당 프로젝트가 정의된 비즈니스 아키텍처(BA)의 목표를 지원하는가? [cite: 5210]
* [cite_start][ ] **데이터 표준**: 전사 데이터 표준 사전 및 명명 규칙을 준수하여 설계되었는가? [cite: 5210]
* [cite_start][ ] **인터페이스 표준**: 시스템 간 통합 시 정의된 표준 메시지/프로토콜을 사용하는가? [cite: 5210]
* [cite_start][ ] **기술 스택**: 인프라 및 플랫폼 아키텍처에서 승인한 소프트웨어/하드웨어 목록을 사용했는가? [cite: 5210]

## 4. EA 성과 관리 (Performance Management)
[cite_start]EA 도입 효과를 정량적으로 증명하여 지속적인 투자를 이끌어냅니다. [cite: 5210]

* [cite_start]**IT 자산 가시성**: 전사 IT 자산 현황을 100% 파악하고 관리하는 수준 측정. [cite: 5210]
* [cite_start]**재사용률 (Reuse Rate)**: 표준 컴포넌트 및 API 재사용을 통한 개발 시간 및 비용 절감액. [cite: 5210]
* [cite_start]**영향도 분석 속도**: 시스템 변경 시 파급 효과를 분석하는 데 소요되는 시간 단축률. [cite: 5210]
